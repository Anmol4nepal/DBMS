DATABASE MANAGEMENT SYSTEM

  1. DATA

    Data refers to the raw facts and figures collected from various sources. These could be numbers, text, symbols, or even multimedia files. Data alone does not carry much meaning until it is processed and interpreted in a specific context. For example, a set of numbers like '24, 30, 18' could represent anything from temperatures to ages unless we are told what they refer to. Once organized and given context, data becomes information â€” something useful for making decisions.

    Types of data used in databases include:

    • Numeric Data: Includes integers and floating-point numbers used in mathematical operations.
    • Text or Character Data: Letters, words, or symbols used for storing names, addresses, etc.
    • Date/Time Data: Used for tracking dates, times, and timestamps in records.
    • Boolean Data: Logical values such as TRUE or FALSE, used for conditional operations.

  2. DATABASE

    A database is a systematic collection of data that is stored electronically in a structured format. It allows data to be easily accessed, managed, updated, and organized. Instead of using paper or separate files, databases centralize all relevant information and make it available to users and software applications.

    Common types of databases include:

    • Relational Databases: Store data in tables with rows and columns. Each table represents an entity, and relationships are established using keys. SQL is the primary language used.
    • NoSQL Databases: Suitable for large-scale data storage. These use document, key-value, graph, or column-family models instead of tables.
    • Hierarchical Databases: Organize data in a tree-like structure, where each record has a single parent and possibly many children.
    • Network Databases: Similar to hierarchical but allows more complex relationships, including many-to-many.

  3. SOFTWARE / SYSTEM (Domain-Specific)

    Domain-specific systems are software solutions built to serve particular business or organizational needs. These systems often rely on databases to store, retrieve, and manage data. They simplify daily tasks by automating and organizing information efficiently.

    • Database Management System (DBMS): The software that allows users to define, manipulate, retrieve, and manage data in databases.
    • Accounting System: Keeps track of financial transactions, budgeting, invoices, tax calculations, and reporting.
    • Attendance System: Monitors employee or student attendance, logs time-in/out, and generates reports on absences or leaves.

  4. SOFTWARE DEVELOPMENT (Database-Focused)

    Developing software that uses databases requires thoughtful planning, design, and maintenance. A strong understanding of data relationships and flow is essential for building scalable and secure applications.

    • Database Design: Involves creating a detailed model of how data is structured and related. This includes defining tables, relationships, keys, and constraints to ensure data integrity.
    • Database Administration: A Database Administrator (DBA) is responsible for installing and configuring DBMS software, setting up user access, maintaining performance, managing backups, and ensuring data security.

  5. TOOLS

    To interact with databases effectively, various tools and concepts are used. These range from basic operations like creating tables to advanced features like securing sensitive data or optimizing performance.

    • Server: A computer system that hosts the database software and services multiple clients or users simultaneously.
    • SQL (Structured Query Language): The standard language used to communicate with and manipulate relational databases.
      • DDL (Data Definition Language): Commands like CREATE, ALTER, and DROP to define or modify schema.
      • DML (Data Manipulation Language): Commands such as INSERT, UPDATE, and DELETE to manage data.
      • DQL (Data Query Language): Uses the SELECT statement to retrieve data from one or more tables.
      • DCL (Data Control Language): Includes GRANT and REVOKE to control user access to data.
      • SQL Joins: Used to combine rows from two or more tables based on related columns.
    • CRUD Operations:
      • Create: Adds new records to the database.
      • Read: Retrieves existing data for viewing or processing.
      • Update: Modifies existing records.
      • Delete: Removes data permanently.
    • Keys in Database:
      • Primary Key: Uniquely identifies each record in a table.
      • Foreign Key: A field that links to the primary key of another table, establishing relationships.
      • Composite Key: A combination of two or more columns used to uniquely identify a record.
    • Constraints: Rules enforced on data columns to maintain accuracy and integrity, such as NOT NULL, UNIQUE, CHECK, and DEFAULT.
    • Anomalies: Problems that arise due to poor database design, such as redundant data (insertion anomaly), lost data (deletion anomaly), and inconsistent data (update anomaly).
    • Normalization: A process of organizing data to reduce redundancy and improve integrity. It involves dividing a database into smaller tables and defining relationships between them. Common forms include 1NF (eliminate repeating groups), 2NF (remove partial dependencies), and 3NF (remove transitive dependencies).
    • Database Security: Measures and techniques to protect data from unauthorized access, manipulation, or loss. This includes access control, encryption, backups, user authentication, and auditing.
